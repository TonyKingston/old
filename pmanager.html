Combinatorics - Creating a Password Manager

This assignment consisted of building a program to fufil the following core objectives

  • Allow the creation and storage of passwords by encrypting some user input
  • Allow users to authenticate themselves by inputting a valid username and password
  • Perform a strength anaylsis on a list of randomly generated passwords

I will be focusing on the third problem as it was the main challenge of the assigment and serves as the most interesting point of disucssion.

First of all, I'd like to explain the rules behind the generation of the passwords. I create 20000 passwords overall which are split into two categories of 10000 each. The first set allows passwords to consists of 10 lowercase letters which can be repeated. The second set doesn't allow repeats, but opens up the number of allowed characters to the extended ASCII set of 256. For each set, the first 100 passwords are one character in length, the second 100 are two characters long, the third three characters long and so on up until they reach 100 characters long. The input data in encrypted by using the Collatz conjecture which is defined below.

You can read this simply as "if the number is ever, divide by two, else triple it and add one". Starting with an offset of zero, we convert the first character in the password into a decimal value which is passed into the Collatz function. The number of iterations required to first reach one becomes the encrypted version of this character. The calculated value then becomes an offset for the next character, being added to it's decimal value before being put through the function. Once all characters have been encrypted, we have our fully encrypted password.
