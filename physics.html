Simulating Physics in 3D Space

The brief for the university project came in two parts. The first half involved creating an obstacle course style environment for a player to move around in, demonstrating the application of various physics principles. Here's an overview of some of the things that it involved:

  • Used semi-implicit Euler for integrating velocity and acceleration.
  • Collision detection is split into broad and narrow phases, with sweep-and-prune being used for the latter. Collisions are detected between various pair of collision volumes: OBB/OBB, Sphere/Sphere, OBB/Sphere and Capsule/Capsule. Raycasts can be performed against these volumes. Collisions are resolved using projection and impulse resolution where friction is also applied. I made use of layer masks to filter which objects can collide with which and what objects should be raycastable.
  • I used a quadtree to spatially partion the scene. In the broadphase, only nodes which have active objects are considered for collison detection, removing a lot of unnecessary checks between static objects.
  • A pushdown automaton was used to manage the state of the game, allowing the user to initiate a game, pause the game and exit the game safely.
  • I've implemented constraints to lock the movement and orientation of an object in a particular set of axes. The pendulum uses this since I didn't what it to possibly spin out of control if the player collided with enough force.

The second half involved creating a maze where the player needs to move around and collect coins for points. They play against an AI, which needs to pathfind around the level and also collect coins.

  • The AI uses the A* pathfinding algorthim to navigate the level. The game reads in data from file which it then uses to construct a set of valid nodes to traverse.
  • The AI makes use of a state machine to switch between pursuing the player or finding a coin. They are also capable of firing out rays which they use to pin point the location of a coin.
